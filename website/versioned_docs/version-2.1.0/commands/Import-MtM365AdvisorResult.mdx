## SYNOPSIS

Imports M365Advisor test result JSON files from disk into PowerShell objects.

## SYNTAX

```powershell
Import-MtM365AdvisorResult [-Path] <String[]> [-ProgressAction <ActionPreference>] [<CommonParameters>]
```

## DESCRIPTION

Loads one or more M365Advisor test result JSON files and returns them as an array
of single-tenant M365AdvisorResults objects.
This is the standard way to load
previously saved results for use with Merge-MtM365AdvisorResult, Compare-MtTestResult,
or any future command that operates on result objects.

If a loaded JSON file contains a multi-tenant merged format (i.e.
a "Tenants"
array from a prior Merge-MtM365AdvisorResult call), each tenant is automatically
expanded into a separate result object.

Accepts file paths, glob patterns, or directory paths.
When a directory is
provided, it auto-discovers TestResults-*.json files inside it.

## EXAMPLES

### EXAMPLE 1

```powershell
# Load a single result file
Import-MtM365AdvisorResult -Path ./production.json
```

### EXAMPLE 2

```powershell
# Load all JSON files matching a glob
Import-MtM365AdvisorResult -Path ./results/*.json
```

### EXAMPLE 3

```powershell
# Load from a directory (auto-discovers TestResults-*.json)
Import-MtM365AdvisorResult -Path ./test-results/
```

### EXAMPLE 4

```powershell
# Pipe into Merge for a multi-tenant report
Import-MtM365AdvisorResult -Path *.json | Merge-MtM365AdvisorResult | Get-MtHtmlReport | Out-File report.html
```

## PARAMETERS

### -Path

One or more paths to JSON result files, glob patterns, or directories.
- File path:  ./production.json
- Glob:       ./results/*.json
- Directory:  ./results/  (discovers TestResults-*.json inside)
