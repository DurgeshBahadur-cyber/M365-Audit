IP-based named locations used in CA 'trusted location' exclusions can be spoofed via VPN or proxy. Relying on IP reputation as a trust signal without combining it with device compliance or MFA creates an authentication bypass for any attacker who can route through a trusted IP range.

#### Remediation action:

IP-based trusted locations can be spoofed via VPN or proxy. Consider Global Secure Access compliant network checks or country-based locations for stronger assurance

Configure via: Entra admin center > Protection > Conditional Access > Named locations
