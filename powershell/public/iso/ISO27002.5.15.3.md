Standard MFA (TOTP, push notification, SMS) is bypassed by AiTM phishing toolkits (e.g., Evilginx, Modlishka) that relay authentication in real time and capture session cookies. Phishing-resistant MFA (FIDO2, Windows Hello, certificate-based) is cryptographically bound to the origin and cannot be relayed by an attacker-controlled proxy.

#### Remediation action:

Create a CA policy: Target admin roles > Grant > Require authentication strength > Phishing-resistant MFA

Configure via: Entra admin center > Protection > Conditional Access
